# 帖子全流程完善方案

> 项目：任务搭子（校园搭子平台）  
> 当前状态：已实现发帖、浏览、标记完成的基础骨架，缺失招募、进行、评价等核心环节  
> 目标：打通「发布 → 招募 → 进行 → 完成/放弃 → 互评」完整闭环

## 二、帖子状态机

```
招募中 ──[搭子加入]──→ 进行中 ──[发布者提交证据]──→ 待评价 ──[双方互评完成]──→ 已完成
   ↑                      │
   └──[搭子退出]──────────┤
                          └──[发布者放弃]──→ 已放弃
```

| 状态 | 含义 |
|------|------|
| `招募中` | 刚发布，等待搭子加入 |
| `进行中` | 已有搭子，任务执行中 |
| `待评价` | 发布者已提交证据，等双方互评 |
| `已完成` | 双方均已评价，流程结束，结算积分 |
| `已放弃` | 发布者主动放弃，扣除罚分 |

## 三、功能清单与实现步骤

### 功能 1：搭子加入 / 退出

**目标**：将帖子从 `招募中` 推进到 `进行中`，记录搭子身份。

#### 后端

1. **新增字段**（`db.js`）
   - `posts` 表增加 `buddyId VARCHAR(64) NULL` 外键指向 `users(id)`
   - 通过 ALTER 语句兼容已有数据

2. **新增接口**（`server.js`）

   ```
   POST /api/posts/:id/join
   Body: { userId }
   逻辑：
     - 检查帖子状态为 招募中
     - 检查 userId ≠ publisherId（不能自己搭自己）
     - 事务内：UPDATE posts SET status='进行中', buddyId=userId, buddyName=用户昵称
     - 返回更新后的帖子
   ```

   ```
   POST /api/posts/:id/quit
   Body: { userId }
   逻辑：
     - 检查帖子状态为 进行中
     - 检查 userId == buddyId
     - 事务内：UPDATE posts SET status='招募中', buddyId=NULL, buddyName=''
     - 返回更新后的帖子
   ```

#### 前端

3. **帖子详情页**（`post-detail`）
   - 根据当前用户身份动态显示按钮：
     - 游客（非发布者非搭子）且状态为 `招募中`：显示「加入任务」
     - 当前用户是搭子且状态为 `进行中`：显示「退出任务」
     - 发布者：不显示加入/退出
   - 加入成功后刷新页面，更新状态和搭子昵称显示

4. **api.js** 新增：
   ```js
   joinPost(id, userId)   // POST /api/posts/:id/join
   quitPost(id, userId)   // POST /api/posts/:id/quit
   ```

### 功能 2：提交任务证据

**目标**：发布者在任务进行中上传证据，推进到 `待评价`。

#### 后端

1. **新增接口**（`server.js`）

   ```
   POST /api/posts/:id/evidence
   Body: { type, value, userId }
   逻辑：

     - 检查 userId == publisherId（只有发布者提交证据）
     - 插入 evidences 表（type: '图片'|'文字', value: 内容/URL）

     - 返回新建的证据记录
   ```

#### 前端

2. **帖子详情页** 为发布者新增「提交证据」入口（状态为 `进行中` 时显示）
3. **新建提交证据弹窗/页面**（可用 `wx.showModal` 简单实现文字证据，图片上传见功能3）
4. **api.js** 新增：
   ```js
   submitEvidence(postId, payload)   // POST /api/posts/:id/evidence
   ```

### 功能 4：提交互评

**目标**：双方（发布者和搭子）各提交一次评价，均完成后推进到 `已完成` 并结算积分。

#### 后端

1. **新增字段**（`db.js`）
   - `evaluations` 表增加 `fromId VARCHAR(64)` 记录评价人ID
   - `posts` 表增加 `publisherEvaluated TINYINT(1) DEFAULT 0`、`buddyEvaluated TINYINT(1) DEFAULT 0`

2. **新增接口**（`server.js`）

   ```
   POST /api/posts/:id/evaluate
   Body: { userId, score, content }
   逻辑：
     - 检查帖子状态为 待评价
     - 检查 userId 为 publisherId 或 buddyId
     - 检查该用户尚未评价（防重复）
     - 插入 evaluations（fromId, fromName, score, content）
     - UPDATE posts 对应评价标志位置 1
     - 若双方均已评价：
         事务内：UPDATE posts SET status='已完成', progress=100
         UPDATE users SET points = points + reward WHERE id = publisherId
         重算 completionRate（见功能6）
     - 返回更新后的帖子
   ```

#### 前端

3. **帖子详情页** 为发布者和搭子各自显示「提交评价」入口（状态为 `待评价` 且本人未评价时）
4. **新建评价表单**（1-5 分 + 文字评语），提交后刷新详情
5. **api.js** 新增：
   ```js
   submitEvaluation(postId, payload)   // POST /api/posts/:id/evaluate
   ```

---

### 功能 5：发布者放弃任务

**目标**：发布者可放弃任务，扣除罚分，帖子标记为 `已放弃`。

#### 后端

1. **新增接口**（`server.js`）

   ```
   POST /api/posts/:id/abandon
   Body: { userId }
   逻辑：
     - 检查 userId == publisherId
     - 检查帖子状态为 招募中 或 进行中（已完成/已放弃 不可再操作）
     - 事务内：
         UPDATE posts SET status='已放弃'
         UPDATE users SET points = MAX(0, points - penalty) WHERE id = publisherId
         重算 completionRate（见功能6）
     - 返回更新后的帖子
   ```

#### 前端

2. **帖子详情页** 为发布者显示「放弃任务」按钮（`招募中`/`进行中` 状态时）
3. 点击后弹出二次确认（`wx.showModal`），避免误触
4. **api.js** 新增：
   ```js
   abandonPost(postId, userId)   // POST /api/posts/:id/abandon
   ```

### 功能 6：completionRate 自动重算

**目标**：每次任务终态（`已完成`/`已放弃`）变更时，自动更新发布者的 `completionRate`。

#### 后端

在 `db.js` 或 `server.js` 中新增辅助函数：

```js
async function recalcCompletionRate(userId) {
  const [row] = await query(
    `SELECT
       COUNT(*) AS total,
       SUM(status = '已完成') AS done
     FROM posts WHERE publisherId = ?`,
    [userId]
  );
  const rate = row.total > 0 ? Math.round((row.done / row.total) * 100) : 0;
  await query('UPDATE users SET completionRate = ? WHERE id = ?', [rate, userId]);
}
```

在 `complete`（功能4触发）和 `abandon`（功能5触发）的事务中调用此函数。

### 功能 7：发布者编辑帖子

**目标**：在帖子仍处于 `招募中` 时，发布者可修改内容。

#### 后端

`PUT /api/posts/:id` 已存在，补充校验：
- 检查 `userId == publisherId`（需前端传入 userId）
- 只允许状态为 `招募中` 时编辑（进行中及以后不可改）

#### 前端

1. **个人主页** 或 **帖子详情页** 在 `招募中` 状态下为发布者显示「编辑」入口
2. 复用 `publish` 页面，通过路由参数 `?postId=xxx` 区分新建/编辑模式
3. 编辑模式下预填表单，提交时调用 `updatePost(id, payload)`

---

### 功能 8：服务端 Token 鉴权

**目标**：防止任意用户冒充他人操作。

#### 后端

1. 在 `POST /api/auth/login` 改为返回真实 JWT：
   ```
   npm install jsonwebtoken
   payload: { userId, openid }
   secret: 从 .env 读取 JWT_SECRET
   ```
2. 新增中间件 `authMiddleware`，解析 `Authorization: Bearer <token>` 头，将 `userId` 注入 `req.userId`
3. 所有写操作路由（发帖、加入、放弃、评价等）挂载此中间件，不再依赖 body 中的 `userId`

#### 前端

4. 登录成功后将 token 存入 `wx.setStorageSync('token', token)`
5. `api.js` 的 `request` 函数统一在 header 中附加 `Authorization: Bearer ${token}`

```sql
-- 状态枚举扩展（MySQL 无 ENUM 约束，应用层控制即可）
-- 新增有效值：'待评价'、'已放弃'
```

所有 ALTER 语句按现有 `db.js` 风格，放在 `createTables` 之后的兼容性 ALTER 块中，确保旧库升级时不报错。
